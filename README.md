# CUDA Practice

A repository to practice CUDA programming using CMake and Make. Each `.cu` file in `src/` is compiled into its own executable and run automatically.

## Device Specifications

- **GPU**: NVIDIA Quadro T1000
- **CUDA Version**: 12.8
- **C++ Compiler**: GCC 11.4.0
- **CMake Version**: 3.18+
- **OS**: Windows Subsystem for Linux (WSL2)

## Project Structure

```
.
├── src/
│   └── hello_world.cu   # starter CUDA kernel
├── build/               # generated by make build (gitignored)
├── CMakeLists.txt
└── Makefile
```

## Makefile Commands

| Target | Description |
|---|---|
| `make build` | Build all `.cu` files in `src/` and run each one |
| `make build <file.cu>` | Build and run a single file (e.g. `make build hello_world.cu`) |
| `make clean` | Remove build artifacts and Python cache files |
| `make clean-all` | `clean` + remove the Poetry virtual environment |
| `make configure` | Install Python dependencies via Poetry |

### Examples

```bash
# Build and run all CUDA programs
make build

# Build and run a single program
make build hello_world.cu

# Clean build directory
make clean
```

## Adding a New CUDA Program

1. Create a new `.cu` file in `src/`, e.g. `src/vector_add.cu`.
2. Run `make build vector_add.cu` — CMake picks it up automatically via `file(GLOB SOURCE_FILES src/*.cu)`.

## Current Programs

### `hello_world.cu`

Launches a kernel with 2 blocks × 4 threads (8 total). Each thread prints its block and thread index.

```
Hello from block 0, thread 0
Hello from block 0, thread 1
...
Hello from block 1, thread 3
```
