# CUDA Practice

A repository to practice CUDA programming using CMake and Make. Each `.cu` file in `src/` is compiled into its own executable and run automatically. All programs are linked against [libwb](https://github.com/abduld/libwb), which provides CUDA error checking, argument parsing, timing, and I/O utilities.

## Device Specifications

- **GPU**: NVIDIA Quadro T1000
- **CUDA Version**: 12.8
- **C++ Compiler**: GCC 11.4.0
- **CMake Version**: 3.18+
- **OS**: Windows Subsystem for Linux (WSL2)

## Project Structure

```
.
├── extern/
│   ├── libwb/           # git submodule — CUDA utility library (built once via make configure)
│   └── ECE408_SP25/     # git submodule — ECE408 course labs (SP25)
├── src/
│   └── <name>/
│       └── <name>.cu    # each subdirectory is one CUDA program
├── build/               # generated by make build (gitignored)
├── CMakeLists.txt
└── Makefile
```

## Getting Started

```bash
# 1. Clone with submodules
git clone --recurse-submodules <repo-url>

# 2. Install Python deps, init libwb submodule, and build libwb once
make configure

# 3. Build and run all programs
make build
```

If you cloned without `--recurse-submodules`, `make configure` will handle the submodule init automatically.

## Makefile Commands

| Target | Description |
|---|---|
| `make configure` | Install Python deps via Poetry, init `extern/libwb` submodule, and build libwb (skipped if already built) |
| `make build` | Build all `.cu` files in `src/` and run each one |
| `make build <file.cu>` | Build and run a single file (e.g. `make build hello_world.cu`) |
| `make debug <file.cu>` | Build with `-g -G` for use with `cuda-gdb` |
| `make profile <file.cu>` | Profile with Nsight Systems — saves `.nsys-rep` to `build/profiles/` |
| `make ncu <file.cu>` | Kernel-level analysis with Nsight Compute — saves `.ncu-rep` to `build/profiles/` |
| `make libwb` | Rebuild libwb manually (e.g. after updating the submodule) |
| `make clean` | Remove build artifacts and Python cache files |
| `make clean-all` | `clean` + remove the Poetry virtual environment |

### Examples

```bash
# Build and run all CUDA programs
make build

# Build and run a single program
make build hello_world.cu

# Debug a specific program
make debug hello_world.cu

# Profile with Nsight Systems
make profile hello_world.cu

# Clean build directory
make clean
```

## Adding a New CUDA Program

1. Create a subdirectory under `src/`, e.g. `src/vector_add/`.
2. Add your kernel as `src/vector_add/vector_add.cu`.
3. Run `make build vector_add.cu` — CMake picks it up automatically.

You can use `#include <wb.h>` in any `.cu` file to access libwb utilities (see below).

## libwb

[libwb](https://github.com/abduld/libwb) is included as a git submodule at `extern/libwb` and built once as a static library (`extern/libwb/build/libwb.a`). It is linked automatically into every CUDA target — it is **never recompiled** during `make build`.

Key APIs (all via `#include <wb.h>`):

| Module | What it provides |
|---|---|
| `wbCUDA` | `wbCheck(cudaCall)` — assert on CUDA errors |
| `wbArg` | `wbArg_read(argc, argv)` — parse `-e`, `-i`, `-o`, `-d` flags |
| `wbTimer` | `wbTime_start` / `wbTime_stop` — GPU and CPU timers |
| `wbImport` | Read float arrays and CSV datasets from files |
| `wbExport` | Write results to files |
| `wbImage` | Load and save PPM images |
| `wbSolution` | Compare output against a reference solution |
| `wbLogger` | Structured JSON logging (ECE408 autograder compatible) |
